# AES-256-GCM 加解密是怎么工作的？

> 一篇写给非技术人员的加解密科普

## 先讲个故事

想象你要寄一封密信给朋友。

你们事先约定了一个暗号（密码），比如"芝麻开门"。你用这个暗号把信的内容打乱变成一堆乱码，塞进信封寄出去。朋友收到后，用同样的暗号把乱码还原成原文。

这就是**对称加密**——加密和解密用的是同一个密码。

我们这个工具用的 AES-256-GCM，就是目前最主流的对称加密方案之一。银行、军方、各大互联网公司都在用它。

---

## 加密过程：三步把明文变成密文

### 第一步：把密码"炼"成钥匙

你输入的密码可能是 `hello123`，但加密算法需要一把固定长度（256位）的钥匙。就好比你的密码是一块铁矿石，需要先炼成一把标准尺寸的钥匙。

这个"炼钥匙"的过程叫 **PBKDF2**，它会：

1. 先撒一把**随机盐（salt）**——就像炒菜撒盐，每次撒的量不同，同样的铁矿石也能炼出不同的钥匙
2. 然后反复锤炼 **10 万次**——故意拖慢过程，让想暴力猜密码的人付出巨大代价

> **为什么要随机盐？** 如果没有盐，所有用 `hello123` 的人炼出的钥匙都一样。黑客可以提前算好一张"密码→钥匙"的对照表（彩虹表），直接查表破解。加了随机盐之后，同样的密码每次炼出的钥匙都不同，对照表就废了。

### 第二步：用钥匙加密

拿到钥匙后，算法还会生成一个**随机初始向量（IV）**——你可以理解为"起始位置"。

想象一个密码转盘锁，钥匙决定了转盘的齿轮配置，IV 决定了起始刻度。同样的钥匙 + 不同的起始刻度 = 不同的加密结果。

这样即使你用同一个密码加密同一段文字两次，得到的密文也完全不同。

### 第三步：打包输出

最终输出的密文其实是三样东西拼在一起：

```
┌──────────┬────────────┬──────────────────┐
│ 盐(salt) │ 起始位置(IV) │ 加密后的乱码(密文) │
│ 16字节   │ 12字节      │ 剩余部分          │
└──────────┴────────────┴──────────────────┘
```

然后整体转成一串字母数字（Base64 编码），方便复制粘贴。

> 盐和 IV 不是秘密，可以公开。它们的作用是让每次加密结果不同，而不是保护内容。真正保护内容的是你的密码。

---

## 解密过程：反过来拆

收到密文后，解密就是把上面的过程倒着走：

```
密文（一串Base64字母数字）
  ↓ 解码
拆出：盐 + 起始位置 + 加密后的乱码
  ↓
用 盐 + 你输入的密码 → 炼出同一把钥匙
  ↓
用 钥匙 + 起始位置 → 解开乱码 → 得到原文
```

如果密码不对，炼出的钥匙就不对，解不开——算法会直接告诉你"解密失败"，而不会给你一堆错误的乱码。

---

## GCM 模式的特别之处：防篡改

GCM 的全称是 Galois/Counter Mode。它比普通加密多了一个能力：**验证密文有没有被人改过**。

打个比方：普通加密就像把信锁进保险箱，GCM 加密则是锁进保险箱的同时，在箱子外面贴了一张防伪封条。

如果有人偷偷改了密文里的哪怕一个字符，解密时封条校验就会失败，算法会拒绝解密。这就防止了有人在传输过程中篡改内容。

---

## 为什么说这很安全？

| 你可能担心的事 | 我们是怎么防的 |
|---|---|
| 密码太简单被猜到 | PBKDF2 炼钥匙要跑 10 万次，暴力猜每个密码都很慢 |
| 黑客用对照表(彩虹表)破解 | 每次加密用随机盐，对照表无效 |
| 加密两次结果一样被分析规律 | 每次用随机 IV，同样内容每次密文不同 |
| 密文在传输中被人偷改 | GCM 模式自带防篡改校验 |
| 密码泄露给网站 | 所有计算都在你的浏览器里完成，密码不会发送到任何服务器 |

---

## 一句话总结

**你的密码 + 随机佐料 → 炼成一把独一无二的钥匙 → 锁住你的信息 → 贴上防伪封条 → 输出一串密文。**

只有知道密码的人，才能用同样的佐料炼出同一把钥匙，撕开封条，还原信息。

---

## 256 是什么意思？

AES-**256** 表示钥匙的长度是 256 位（bit），也就是一个由 0 和 1 组成的 256 位数字。

可能的钥匙组合数是 2 的 256 次方，大约是：

> 115,792,089,237,316,195,423,570,985,008,687,907,853,269,984,665,640,564,039,457,584,007,913,129,639,936

这个数字比宇宙中所有原子的数量还要多。即使全世界所有的计算机联合起来暴力穷举，到太阳燃尽也试不完。

所以，只要你的密码本身不太简单（别用 `123456`），这个加密方案在当前技术条件下是不可破解的。
